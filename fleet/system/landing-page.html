Design principles

The philosophy behind every architectural decision.

Inference is a resource to be allocated, not a constant.

Each agent and each sub-task uses the minimum inference tier that produces acceptable quality. Local Qwen for parsing, cloud Haiku for synthesis, Sonnet for deep research, Opus for architectural reasoning. 90% of execution is pure Python at $0.

Lumina orchestrates, never executes bulk work.

Maximum ~5 tool calls per user request. Complex work is delegated to sub-agents via Nexus inbox. Lumina plans and monitors. Sub-agents execute.

Custom over native OpenClaw.

IronClaw lacks native multi-agent communication. Python sub-agents aren't IronClaw agents. Our custom inbox (Nexus) provides more control over inference routing and cost than native sessions_send ever would.

No peer-to-peer agent messaging.

All communication routes through Lumina via Nexus. Prevents runaway delegation loops, keeps token costs predictable, maintains a single audit trail.

Tier 3 — pure Python

API calls, retries, state management, Nexus routing. No LLM involvement. Handles 90% of Axon's work, all Sentinel health checks, all Nexus message routing.

Tier 2 — local Qwen

Qwen 2.5:7b on VM901 GPU via LiteLLM. $0 inference. Used for parsing, label naming, description writing, per-source summarization in Seer, Vigil formatting.

Tier 1 — cloud

Haiku + Sonnet via OpenRouter. Lumina's orchestration, Seer deep synthesis ($2-5/report), Meridian market analysis. The thinking that requires judgment.

Message envelope

from_agent: lumina | axon | vigil | ...
to_agent: lumina | axon | * (broadcast)
message_type: work_order | result | escalation | status | heartbeat
priority: critical | urgent | normal | low
payload: { ... } (JSON, schema per type)

Fleet agents — what each one does

All agents run on CT310. They communicate via Nexus and publish HTML via Apache.

Axon — Work Queue Manager

systemd service · polls Nexus every 60s · direct psycopg2 to CT300

Receives work_order messages from Lumina via Nexus. Decomposes into atomic API calls. Executes bulk Plane operations (create issues, labels, states) or Gitea operations (create repos, push files). Creates tracking issues in The Plexus. Returns results via Nexus. Queries Engram for prior context before acting.

Vigil — Briefing Agent

IronClaw routines · 6:45 AM + 3:50 PM weekdays · Qwen for formatting ($0)

Gathers data from 15 sources in parallel (ThreadPoolExecutor): weather, commute, news (3 topics), sports, crypto, Plex, Jellyseerr, cluster health, fun fact, SETI, reflection. Generates HTML dashboard at /briefing/. Matrix message is a link + one-line summary.

Sentinel — Ops Monitoring

10 health operations · HTML status page · auto-refreshes every 5 min

Runs health checks across all services (Prometheus, Plex, Gitea, VM901, Ollama). Generates /status/ HTML with green/amber/red per service. Auto-updates after system-snapshot, self-health, plex-health operations. Checks 12 services.

Seer — Research Engine

8-step pipeline · 3 tiers · sanitization on all fetched content

Query planner (Qwen) → SearXNG search → source ranker → 3-layer injection sanitization → per-source summarization (Qwen, isolated, canary-validated) → gap analysis loops → synthesis (Haiku or Sonnet) → HTML + Markdown publish. Deep tier includes bias evaluation, comparison matrix, 3 Sonnet passes.

Mr. Wizard — Deep Reasoning

Auto tier (Sonnet ~$0.10) · Gated tier (Opus via Claude OAuth)

Convenes the Obsidian Circle: Architect Arcane (systems design), Skeptic Seer (devil's advocate), Keeper of Operations (3 AM runbook), Wandering Fool (lateral thinking). Four sequential perspective passes, then Mr. Wizard synthesizes. Results in Engram + Nexus result to Lumina.

Engram — Semantic Memory

sqlite-vec · nomic-embed-text · 1536-dim · 3 layers

Three memory layers: knowledge_base (infrastructure facts, decisions), patterns (conventions, naming rules), activity_journal (timestamped agent actions). Embedding via LiteLLM → nomic-embed-text on Ollama (VM901). Text search fallback when embedding fails.

Using Engram from IronClaw

engram_query("how were Plane labels created")
→ Returns top-K semantically similar KB entries

engram_store(key="decision/auth", content="...", layer="patterns")
→ Embeds + stores to sqlite-vec DB

engram_journal(agent="axon", action="created labels", outcome="5 done")
→ Activity log entry with embedding

engram_conventions(topic="naming")
→ Returns relevant patterns as markdown

engram_recent(hours_back=24, agent_filter="lumina")
→ Recent journal entries for context

Vector — dual-mode dev loops

Standalone (open source, zero infra) or integrated (full Lumina stack). Same core loop, different backends.

Interface contracts (Phase 1 — complete)

Four abstract backends. Core loop never imports implementations directly.

InterfaceStandaloneIntegrated
StateBackendSQLitePlane (The Plexus)
MessageBusstdout + logNexus (psycopg2)
MemoryStore./memory/*.mdEngram (sqlite-vec)
CostGateHard cap $5Nexus → Lumina approval

8-phase loop (Phase 2 — building)

1. INTAKE ← work order (CLI or Nexus)
2. PLAN ← LLM decomposes into tasks
3. EXECUTE ← write code, apply changes
4. TEST ← run test command, eval output
5. REVIEW ← MemoryStore + LLM judgment
6. ITERATE ← fix or advance
7. COMPLETE ← StateBackend.complete_task()
8. LEARN ← MemoryStore + CostGate

Escalates (not loops forever) on: 3 consecutive failures · architectural ambiguity · budget exhausted · scope creep
